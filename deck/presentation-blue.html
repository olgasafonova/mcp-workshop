The Problem

AI can answer questions. But it can't take action.

Most AI assistants today can summarize documents, generate text, answer questions.

But they can't create a Miro board. Update your roadmap. Control your browser. Edit files on your machine.

3 / 19
Key Difference

RAG vs MCP

RAG (Retrieval)

↓ ↓ ↓

Information flows IN to the AI

AI retrieves and reads. Still just a smarter search bar.

MCP (Action)

↑ ↑ ↑

Commands flow OUT from the AI

AI takes action in your apps. Becomes a collaborator.

4 / 19
The Standard

MCP is USB-C for AI

Model Context Protocol is a standard that lets AI assistants connect to your apps.

Before USB-C

Every device had its own cable. Printers, cameras, keyboards: all different.

After USB-C

One standard, everything connects. MCP does the same for AI.

Tips

Using MCP effectively

You're giving AI a toolbox. Be specific about which app to use.

Good prompts

  • "Use the wiki server to search for auth docs"
  • "Create a Miro sticky saying 'Review API'"
  • "Use Playwright to screenshot the login page"

Less effective

  • "Find some info about authentication"
  • "Make a note somewhere"
  • "Open a website"

Structure

Server architecture

server/
├── main.go              # Entry point, server setup
├── tools/
│   ├── definitions.go  # What can it do? (schemas)
│   └── handlers.go     # How does it do it? (logic)
└── client/
    └── client.go       # API client for the app

Minimal server needs: Tool definition + Handler + API client

Summary

Key takeaways

  • Explorers: MCP lets AI take action, not just answer questions. Works with cloud and local apps.
  • Users: Setup is simple. Download, configure, restart. Start with GLEIF or Playwright.
  • Builders: Start small. My first server had 5 tools. By the fourth, I had patterns figured out.
